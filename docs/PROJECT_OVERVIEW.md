# Wordle 덱 공유 사이트

## 프로젝트 개요

Wordle 게임에서 사용하는 단어 덱을 업로드하고 공유할 수 있는 플랫폼입니다. **회원가입 없이** 누구나 자신만의 단어 세트를 만들어 URL로 공유하고, 다른 사용자들이 해당 덱으로 Wordle 게임을 플레이할 수 있습니다. Discord, Reddit, X 등 외부 커뮤니티를 통한 바이럴을 핵심 성장 경로로 삼아, 덱 제작부터 공유·편집·플레이까지 모든 마찰을 제거하는 것을 최우선으로 합니다.

## MVP (Minimum Viable Product)

### 핵심 기능

- **익명 작성 시스템** (회원가입 불필요)
  - 덱 생성 시 닉네임 + 비밀번호(덱 전용 PIN) 입력
  - 같은 비밀번호로 언제든 수정/삭제 가능
  - Supabase Anonymous Auth 로 세션 기반 신원 자동 발급 (게임 기록/좋아요 연결용)
- **단어 덱 CRUD (생성, 조회, 수정, 삭제)**
  - 덱 정보: 이름, 단어 배열, 썸네일 이미지(선택)
  - 덱 생성/수정/삭제/플레이: 모두 회원가입 불필요
- **덱 URL 공유 기능** (링크 하나로 제작·편집·플레이 모두 가능)
- **공유된 덱으로 Wordle 게임 플레이**
- **덱 검색 및 필터링**
- **내가 만든/편집한 덱 목록** (브라우저 localStorage 기반)
- **좋아요 기능** (익명 세션 기반 중복 방지, 1회 제한)

## 개발 단계

### Phase 1: 기본 기능

1. Next.js 프로젝트 설정
2. Supabase 연동 및 데이터베이스 스키마 구성
3. 익명 작성 시스템 구현 (닉네임+비밀번호 + Supabase Anonymous Auth)
4. 기본 UI 컴포넌트 구성 (shadcn/ui)
5. 덱 CRUD 기능 구현
6. **테스트 환경 설정** (Vitest, Playwright)

### Phase 2: 게임 기능

1. Wordle 게임 로직 구현
2. 커스텀 덱으로 게임 플레이
3. 게임 결과 저장
4. **Critical 테스트 작성** (게임 로직, E2E 플로우)

### Phase 3: 공유 및 커뮤니티

1. URL 공유 기능
2. 검색 및 필터링
3. 좋아요 시스템
4. **Important 테스트 추가** (커뮤니티 기능)

### Phase 4: 최적화 및 배포

1. 성능 최적화
2. Vercel 배포
3. 사용자 피드백 반영
4. **CI/CD 파이프라인 구축** (자동화 테스트)

## 예상 사용 사례

1. **Discord 서버**: 내부 밈/조크 기반 덱을 만들어 채널에 공유
2. **Reddit 서브레딧**: 특정 주제(게임, 영화 등) 단어 덱을 커뮤니티에 공유
3. **팬덤**: 아티스트/작품 관련 덱을 X(Twitter)·팬 포럼에 공유
4. **스터디/교육**: 시험·학습용 단어 덱을 채팅방에 공유
5. **친구 그룹**: 내부 밈·별명 기반 덱을 DM 으로 공유
