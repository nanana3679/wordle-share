# 0008. 추측 입력 자동완성/덱 노출 없음

## Status

Accepted

## Context

워들류 게임에서 "후보 단어 풀"을 어떻게 노출하느냐가 게임 정체성을 결정한다.

- 자동완성 제공 → 비팬도 후보 클릭으로 풀이 가능. IP 게임의 차별점 소실
- 덱 단어 리스트 노출 → 정답 누설
- **노출 없음** → IP 지식 자체가 진입 장벽

본 서비스의 핵심 컨셉은 "이 IP를 아는 사람만 풀 수 있는 커뮤니티 게임". 자동완성은 이 정체성을 무너뜨린다.

## Decision

- 추측 입력 시 **자동완성/덱 단어 리스트 노출 없음**
- 플레이어는 머릿속에서 IP 단어를 떠올려 직접 입력
- 서버 검증: 활성 단어 집합(`deck.words.filter(active)`)에 있으면 추측 처리, 없으면 "이 IP에 없는 단어"로 일반 거절
- 검증은 **라운드 시작 시 활성 단어 ID 스냅샷** 기준 (덱 편집과의 race 방지)

## Consequences

- IP 지식 진입 장벽 형성 — 비팬은 자연스럽게 못 풂 → 커뮤니티 게임 정체성 강화
- 입력 마찰 증가 (생각해서 입력) — 의도된 디자인
- 클라이언트에 단어 리스트가 절대 안 가도록 API 분리 필요 — 게임 페이지 SSR HTML에도 단어 리스트 미포함 ([0012](./0012-ssr-public-deck-pages.md))
- 검색 기능에서도 단어 내용 매칭 차단 → 덱 이름만 매칭
- "후보 보기" 같은 힌트 V2 후보 — 도입 시 retention 증가 가능성과 IP 정체성 약화 균형 재평가
