# 0006. 챌린지 1일 1회 + 데일리 완료 게이트

## Status

Accepted

## Context

챌린지 모드의 접근 제어 후보:

- (a) 무제한 — 정답 누설 위험 + ritual 붕괴
- (b) 1일 1회만 — ritual 보존
- (c) 1일 1회 + **데일리 완료 후 잠금 해제** — 데일리와 챌린지가 서로를 견인

(c)는 **제한 = 게임 디자인 scaffolding**이라는 철학에 부합한다. 시크릿 모드/멀티 디바이스로 우회 가능하지만, 본인 경험을 본인이 망치는 행위. 신뢰는 댓글 스레드/커뮤니티가 담당하고, 플랫폼은 의식(ritual)의 형태만 제공한다.

## Decision

- 챌린지는 `(anon_id, deck_id, date)` 단위로 **1일 1회**만 시작 가능
- **게이트**: 그날 데일리 라운드 완료 (솔브 OR 시도 소진) 후에만 챌린지 잠금 해제
  - `Solve.status = "completed"` 확인
- 챌린지 시드는 데일리와 별개 (`hash(deck_id + date + "endurance")`)
- 시드가 분리되어 있어서 데일리 단어가 챌린지 시퀀스에 포함될 수 있음 — **그대로 둠** (라이트 스포일러 감수)
- 한 라운드에서 시도 소진 시 런 종료, 점수 = 풀어낸 단어 수

## Consequences

- 데일리가 챌린지로 자연 견인 → 양 모드 retention 상호 강화
- 우회 시도자는 본인 경험만 깎임 — 공개 랭킹 없음([0003](./0003-no-public-user-leaderboard.md))이라 우회 인센티브 약함
- 데일리 완료 = 시도 다 써도 인정 → "솔브 못해도 일단 시작은 가능" 흐름 보장
- 라이트 스포일러 (데일리 단어가 챌린지에 등장) 감수. 스코프 작아 무시 가능
- 자정 직후 풀이/탭 동시 진행 시 status 갱신 race는 [0009](./0009-optimistic-locking-with-version.md)로 처리
