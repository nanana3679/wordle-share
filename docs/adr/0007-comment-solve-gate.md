# 0007. 댓글 풀이-게이트 ((deck, date) 단위)

## Status

Accepted

## Context

댓글은 본 서비스의 사회적 hook이다. 위치/게이트 후보:

- 덱 단위 단일 스레드 (게이트 없음) → 정답 누설, retention 동기 약함
- (deck, date) 단위 + **풀이 완료 후 열람** → 스포일러 차단 + 풀이 동기 부여

원조 워들의 댓글 culture(트윗 인증샷)와 유사하게, "풀어야 비로소 보이는" 구조가 retention을 강하게 만든다.

## Decision

- 댓글은 `(deck_id, date)` 단위 스레드. 단일 피드 (덱 상세 페이지 하단), 최신순 + 날짜 헤더로 그룹화
- **게이트 규칙**:
  - **오늘 댓글**: 오늘 데일리 풀이 `completed` 후에만 열람·작성
  - **과거 댓글**: 한 번이라도 풀면 모두 열람 (스포일러는 IP 팬에겐 약한 정보)
  - **제작자**: 게이트 우회 + "제작자" 배지 표시
- 작성 형식: 단순 텍스트 (답글/멘션/이미지 X)
- 본인 삭제: nick+pw로 인증 ([0001](./0001-anon-auth-and-nick-pw-identity.md))
- 신고 버튼

## Consequences

- 댓글 잠금이 풀이 동기 부여 메커니즘으로 작동 (retention)
- 챌린지 점수 자랑이 댓글 스레드에 집중 → 사회적 검증 채널 역할
- 데일리 안 푼 사용자는 오늘 댓글 read도 차단 → enumeration/스크래핑도 자연스럽게 방어
- 댓글 작성 시 nick+pw 입력 필요 → 첫 작성 시점에 명시 입력, 이후 localStorage 자동 채움
- 알림은 in-app 안읽음 카운트만 (푸시/이메일 없음, [기획 9d])
- V2: 댓글 좋아요/이모지 리액션 — MVP 제외
