# 0003. 사용자 점수 랭킹/리더보드 없음

## Status

Accepted

## Context

챌린지 모드는 점수가 나오는 게임 메커닉이다. 자연스럽게 "리더보드 만들자"는 압력이 생기지만, 다음 위협 모델 문제가 있다:

- 챌린지 1일 1회 + 데일리 게이트는 **보안 경계가 아니라 게임 디자인 scaffolding** (intended ritual)
- 시크릿 모드/멀티 디바이스로 우회 가능 — 본인 경험을 본인이 망치는 행위지만, 리더보드가 있으면 우회 인센티브 발생
- 리더보드가 있으면 cheating 1인만 있어도 신호가 100% 무가치화됨
- "신뢰는 댓글 스레드/커뮤니티가 담당. 플랫폼은 ritual의 형태만 제공" 원칙

## Decision

- **공개 사용자 점수 랭킹/리더보드 페이지 없음**
- 챌린지 점수는 **개인 통계**(`UserDeckStats`)와 **댓글 결과 공유** 형태로만 노출
- `UserDeckStats`는 본인 `auth.uid()`에게만 SELECT 허용 (RLS)
- 점수 검증·자랑은 댓글 스레드의 사회적 검증에 위임

## Consequences

- 챌린지 우회 동기 제거 — 우회해도 보여줄 곳이 없음
- 만점 클리어/높은 점수는 댓글 결과 공유 텍스트(이모지+링크)로 자랑 → 커뮤니티(디스코드/단톡방) 단위 사회적 검증
- 콘텐츠(덱) 좋아요 랭킹은 별개 — [0004](./0004-content-likes-vs-user-scores-threat-model.md) 참고
- V2 후보(검토 단계): 본인이 명시 동의한 경우에 한해 닉 + 점수 노출 옵션 — 단 신호 무가치화 위험 재평가 필요
